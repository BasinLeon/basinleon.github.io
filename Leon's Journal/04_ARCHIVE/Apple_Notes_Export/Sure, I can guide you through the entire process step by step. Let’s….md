<div>#### **2.1 OSI Model**</div>
<div>Understanding the OSI (Open Systems Interconnection) model is fundamental for network communication:</div>
<div><br></div>
<div>**Task 4:** Learn about the 7 layers of the OSI model:</div>
<div>- **Layer 1:** Physical Đ Deals with the physical connection between devices.</div>
<div>- **Layer 2:** Data Link Đ Handles data transfer between adjacent network nodes.</div>
<div>- **Layer 3:** Network Đ Determines the best physical path for data to reach its destination.</div>
<div>- **Layer 4:** Transport Đ Ensures complete data transfer.</div>
<div>- **Layer 5:** Session Đ Manages sessions in a network.</div>
<div>- **Layer 6:** Presentation Đ Translates data between the network and application layers.</div>
<div>- **Layer 7:** Application Đ Interacts with the end user directly.</div>
<div><br></div>
<div>**Action Step:** Draw the OSI model and label each layer with its functions and associated protocols (e.g., TCP/IP at Layer 4).</div>
<div><br></div>
<div>**Resources:**</div>
<div>- Use this [OSI Model Tutorial](https://www.geeksforgeeks.org/layers-of-osi-model/) for a detailed explanation.</div>
<div>- Watch &quot[The OSI Model Explained](https://www.youtube.com/watch?v=vv4y_uOneC0)&quot to get a clear understanding.</div>

<div>#### **2.2 Network Topologies**</div>
<div>Network topologies define the layout of the connected devices:</div>
<div><br></div>
<div>**Task 5:** Understand different types of topologies:</div>
<div>- **Star:** All devices are connected to a central hub.</div>
<div>- **Ring:** Each device is connected to two other devices, forming a circular data path.</div>
<div>- **Bus:** All devices share a common backbone to communicate.</div>
<div><br></div>
<div>**Action Step:** Compare and contrast these topologies in terms of reliability, cost, and performance.</div>
<div><br></div>
<div>**Resources:**</div>
<div>- Check out [Network Topologies Explained](https://www.networkcomputing.com/networking/7-network-topologies-understand) for a deeper dive.</div>

<div>#### **2.3 Common Protocols and Ports**</div>
<div>Networking relies on protocols to transmit data:</div>
<div><br></div>
<div>**Task 6:** Learn the most common protocols:</div>
<div>- **HTTP (Port 80):** Used for web traffic.</div>
<div>- **HTTPS (Port 443):** Secure version of HTTP.</div>
<div>- **FTP (Port 21):** Used for file transfers.</div>
<div>- **SMTP (Port 25):** Used for sending emails.</div>
<div><br></div>
<div>**Action Step:** Use a tool like Wireshark to capture and analyze network traffic to see these protocols in action.</div>

<div>#### **2.4 IPv4 and IPv6**</div>
<div>Understanding IP addressing is crucial:</div>
<div><br></div>
<div>**Task 7:** Study the differences between IPv4 and IPv6:</div>
<div>- **IPv4:** 32-bit address, supports 4.3 billion addresses.</div>
<div>- **IPv6:** 128-bit address, supports an almost unlimited number of addresses.</div>
<div><br></div>
<div>**Action Step:** Practice subnetting with IPv4 addresses and understand the structure of IPv6 addresses.</div>
<div><br></div>
