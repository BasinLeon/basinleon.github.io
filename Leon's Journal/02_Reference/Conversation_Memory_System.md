# Conversation Memory System

## Purpose
Strengthen continuity across sessions by storing not just summaries, but also:

- exact user preferences
- important lines and wording
- decisions
- open loops

## Truth Standard
The system should not pretend it remembers every single word automatically.

Instead, it should preserve the parts that matter most in a structured way.

## Memory Layers

### 1. Session note
Captures:

- what happened
- what was built
- what changed
- what comes next

### 2. Reusable reference note
Captures:

- durable frameworks
- project playbooks
- communication patterns

### 3. Conversation memory ledger
Captures:

- exact phrases worth preserving
- user corrections
- decision history
- active open loops

## Verbatim Capture Rule
When Leon says a line or preference that should shape future output, preserve it explicitly.

Example categories:

- "no need to apologize"
- "save everything moving forward to Obsidian"
- "I want this to sound like a revenue architect"

## Recommended Operating Pattern

After meaningful sessions:

1. write the session note
2. update the relevant reference note
3. append any important exact wording to the memory ledger

## Goal
Create reliable continuity without bloating the vault with raw transcripts.
