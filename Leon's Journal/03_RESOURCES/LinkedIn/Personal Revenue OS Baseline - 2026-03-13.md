# Personal Revenue OS Baseline - 2026-03-13

## Core Reframe
- This system is not a nicer CRM.
- It is a personal revenue operating system.
- The real job is:
  - tell me where to focus
  - tell me why it matters
  - tell me what to do next across career, revenue, and relationships

## What Is Already Strong
- Archive and cockpit are correctly split
- LB's Desk is the right metaphor for active work
- Scout is the right recall layer for the bigger universe
- Today Stack is strong because it turns ambiguity into sequence
- Operator language is coherent across the surfaces

## Product Risks To Watch
- Naming density is too high when every panel tries to sound like the star
- Scoring logic is still too implicit
- Pipeline value is too dollar-only for the true use cases
- Scout trust can wobble when live counts and enriched counts are not clearly separated

## Recommended Language Hierarchy
- Tier 1:
  - Desk
  - Scout
  - Today Stack
  - Active Lanes
- Tier 2:
  - Snapshot
  - Radar
  - Brief
  - Data Hub
- Tier 3:
  - pipeline metrics
  - ingest tools
  - saved profiles
  - source details

## Recommended Scoring Logic
- Priority Score should visibly combine:
  - urgency
  - relationship warmth
  - revenue potential
  - strategic value
  - recency of touch
  - next-step clarity

## Value Modes
- Career Value
- Revenue Value
- Strategic Value

## Recommended Operator Modes
- Job Hunt Mode
- Founder Mode
- Revenue Mode
- Relationship Mode

## Why This Matters
- The hard problem is not storing contacts.
- The hard problem is deciding what deserves action now when time, attention, and opportunity are all constrained.
- That is why the system should keep evolving as a decision engine rather than a generic dashboard.
