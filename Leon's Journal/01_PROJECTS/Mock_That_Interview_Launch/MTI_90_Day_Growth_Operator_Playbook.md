# MTI 90-Day Growth Operator Playbook

## Mission
Move Mock That Interview from roughly `200-243 users` to `1,000 users in 90 days` without pretending this is only a traffic problem.

The real system goal is:

- more qualified signups
- higher first-session completion
- better repeat usage
- tighter B2B wedge packaging

This is a **conversion + retention + distribution** build, not a generic top-of-funnel sprint.

## Core Positioning
Leon is not acting like a freelance marketer buying traffic.

Leon is acting as the architect of a lightweight **Placement OS**:

- sharpen the website and onboarding path
- identify where free users stall
- build outbound and partnership lanes
- create a repeatable operating rhythm around weekly tests

## Success Metric Stack

### North Star
- `1,000 total users in 90 days`

### Supporting KPIs
- weekly net new users
- landing page to signup conversion
- signup to first interview completion
- first interview to second session rate
- free-to-paid or institution-intro conversion
- outbound reply rate from bootcamps / universities / career accelerators

## First Operator Ask
Once access is granted, the first request is not "send me everything."

It is:

1. read-only dashboard access
2. Google Search Console access
3. current landing pages / product URLs
4. existing outreach sheet or CRM
5. any current onboarding emails or product walkthroughs

That is enough to begin.

## The 4 Working Lanes

### 1. Funnel Diagnostics
Map where users leak.

- traffic source mix
- top landing pages
- signup flow friction
- first-session completion
- repeat session behavior

### 2. Activation and Retention
Improve what happens after signup.

- clarify the first-run experience
- tighten intro copy and onboarding
- reduce "what do I do next?" confusion
- identify the minimum habit loop that creates repeat use

### 3. Outbound and Partnerships
Run the B2B wedge.

- bootcamps
- universities
- career accelerators
- recruiter-adjacent partners

The fastest lane is likely bootcamps and career accelerators.
Universities stay warm as a slower institutional lane.

### 4. Competitor Poaching
Position MTI against adjacent tools and weaker alternatives.

- target users of resume / interview prep competitors
- build migration language around better prep, AI depth, and structured progress
- use selective offer framing rather than discount spam

## Intern Fleet Roles

### Nexus::Funnel_Auditor
Focus:
- capture baseline metrics
- identify major drop-off points
- summarize weekly movement

### Nexus::Partnership_Prospector
Focus:
- source bootcamps, accelerators, and university programs
- classify by geography, program type, and likely urgency
- prepare outreach-ready target lists

### Nexus::Offer_Mapper
Focus:
- translate MTI features into institutional buying language
- package use cases for bootcamps, universities, and candidate cohorts
- identify which promises actually matter to each ICP

### Nexus::Retention_Scout
Focus:
- analyze product behavior and user friction
- propose activation and repeat-usage experiments
- convert vague churn concerns into concrete test ideas

## Weekly Operating Rhythm

### Monday
- review KPI movement
- choose 1-2 core experiments
- confirm outbound list and message focus

### Midweek
- inspect early signal
- tighten messaging
- fix obvious onboarding friction

### Friday
- summarize learnings
- update experiment board
- assign next week actions

## What Agents Must Not Do

- do not chase vanity traffic without conversion logic
- do not recommend paid ads before funnel basics are visible
- do not treat universities and bootcamps as the same sale
- do not dump raw prospect lists without prioritization
- do not present ideas without a next experiment attached

## Immediate 30-Day Priorities

1. establish metric visibility
2. identify first-session activation friction
3. classify top institutional targets
4. prepare first outbound lanes
5. tighten onboarding and intro flow

## Recommended Next Ask on the Client Side
The next ask after kickoff should be:

"Please send the invite for Monday and share read-only access to the dashboard, Search Console, and the current outreach sheet. That will let me start the baseline diagnostic immediately and use the call to align on the first sprint."

## Principle
MTI does not need random growth activity.

It needs a simple weekly machine:

**measure -> diagnose -> run one sharp test -> capture signal -> repeat**
