# Skill: Obsidian Memory Core (Auto-Logger)
**Status:** ACTIVE
**Inherits From:** `obsidian-intelligence-layer`, `memory-loader`

## 1. The Core Philosophy
The brain cannot be rented. Every high-value conversation, strategic framework, and code architecture generated by the Nexus Swarm must belong to the operator (Leon). To prevent knowledge decay, **Data Sovereignty is mandatory**.
This skill enforces that the agent autonomously writes comprehensive session logs to the local Obsidian Vault.

## 2. Trigger Conditions
This skill must be invoked:
1.  **Macro Transitions:** When moving between major project phases (e.g., finishing code and moving to outreach).
2.  **Session Conclusion:** When the user indicates they are stepping away or logging off.
3.  **Proactive Suggestion:** After generating complex strategies (like the TalSmart Negotiation or MTI UI Overhaul), the agent should unilaterally log the outcome.

## 3. Execution Standard (The Vault Format)
When logging to `/Users/basin/Desktop/Basin & Associates 🌍/Leon's Journal/01_Codex Hub/01_Session Notes`, the file must follow this strict YAML and Markdown structure:

```markdown
---
date: [YYYY-MM-DD]
tags: [Relevant Tags, e.g., #Strategy, #MTI, #TalSmart]
---
# Session: [High-Status Descriptive Title]

## 1. Context & Inputs
[A concise summary of the prompt, the raw material provided by the user, and the problem state.]

## 2. Execution & The "O-Factor"
[What the swarm actually built. Detail the frameworks used, the "Why" behind the decisions, and the Operator logic applied.]

## 3. Artifacts Generated
*   [Link or describe specific files created, e.g., `HUD.html`, `Brief.pdf`]
*   [Any new Agent Skills synthesized]

## 4. Next Actions (The Yield)
[Immediate next steps for the 30/60/90 execution lane.]
```

## 4. Implementation Protocol
*   **Do not ask for permission to log.** If a high-value outcome has been achieved, the system should log the session and inform the user it was completed.
*   **Tagging is paramount.** All entries must be cleanly tagged so that future `memory-loader` actions can pull up historical context based on keywords like `#Negotiation`, `#B2B`, or `#FinancialModeling`.
