# The Multi-Agent Revenue Architecture Stack (LeonOS)

**Operating Principle:** Expand with Gemini. Decide with ChatGPT. Ship with Codex. Orchestrate with Antigravity.

Do not ask four agents to do the same job. Assign job classes to prevent redundancy, flattery loops, and wasted cycles.

## 1. Gemini (Phase 1: Expand)
**Role:** Expansion engine, ideation partner, broad synthesizer.
**When to use:**
- Raw possibilities and brainstorming
- Multiple concepts and wider thinking
- "What am I not seeing?"
- Divergent idea generation and pattern spotting from large contexts.

## 2. ChatGPT / Claude (Phase 2: Tighten)
**Role:** Commercial editor, strategist, pressure-tester, synthesis layer.
**When to use:**
- Deciding what matters and what is fluff.
- Turning messy thinking into clean narratives.
- Tightening positioning and audience calibration.
- *The practical test:* "Is this actually good or just smart-sounding?"

## 3. Codex (Phase 3: Build)
**Role:** Builder, codebase architect, implementation arm.
**When to use:**
- Writing complex Python, React, or database schemas.
- Building dashboards, parsers, classifiers.
- Handling the physical code architecture.

## 4. Antigravity (Phase 4: Orchestrate)
**Role:** Local operating system layer, terminal operator, file manager, repo executor.
**When to use:**
- Editing files locally, creating folders, updating READMEs.
- Pushing to GitHub via terminal.
- Triggering browser tasks and extracting raw data transcripts.
- Organizing assets and routing outputs between systems.

---

## Operating Rhythm (Example: Processing Transcripts)
1. **Antigravity (Me):** Extracts the raw Zoom/ChatGPT transcript via browser automation and stores it locally.
2. **Gemini:** Reads the raw text and pulls 5 divergent themes and macro-patterns.
3. **ChatGPT:** Takes those 5 themes, cuts the fluff, and turns them into 1 actionable commercial insight.
4. **Antigravity / Codex:** Takes that 1 insight, updates the `basin-nexus` README, and commits the code.
