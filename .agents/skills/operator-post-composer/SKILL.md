# Operator Post Composer

## Overview

Build posts that signal execution ownership, not consultant theater.

## When to Use

- Writing a LinkedIn or X post for role positioning.
- Translating messy ideas into one clear market point.
- You need a post that sounds senior and calm.
- You want practical, outcome-first language.

## Positioning Lens

Default to this identity unless user overrides:

- internal operator
- cross-functional systems owner
- focused on predictable execution

## Post Structure

Use 6 blocks max:

1. Hook (1 line)
2. Problem (1-2 lines)
3. Pattern (2-3 bullets or short lines)
4. Reframe (1 line)
5. Outcome lens (1-2 lines)
6. Soft close (1-2 lines)

## Rules

- Keep under 140 words unless user asks longer.
- One thesis per post.
- Prefer concrete terms: ownership, routing, handoffs, cycle time.
- Avoid hype words: game-changing, revolution, paradigm shift.
- End with signal, not desperation.

## Output Modes

Provide these by default:

- `publish now` version
- `more casual` version
- `more direct` version

## Hashtag Policy

- 0-2 hashtags max for LinkedIn.
- No hashtags for X unless user asks.
