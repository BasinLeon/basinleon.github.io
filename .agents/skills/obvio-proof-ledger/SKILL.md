# Obvio Proof Ledger

## Purpose
Turn messy activity into durable private proof. The output should help the user remember what happened, explain why it mattered, and reuse the evidence later in reports, reviews, or executive updates.

## Default Format
Use this structure unless the user asks otherwise:

```markdown
YYYY-MM-DD - CATEGORY / MOTION

WHAT:

IMPACT:

KPI:

NEXT:
```

## Categories
Choose the closest category:

- `OUTREACH ACTIVITY`
- `RESEARCH & INTEL`
- `TOOLS & SYSTEMS BUILT`
- `STAKEHOLDER ALIGNMENT`
- `RELATIONSHIPS`
- `INTERNAL VISIBILITY`
- `CRM ARCHITECTURE`
- `PIPELINE EXECUTION`
- `GTM PLAYBOOK`
- `PUBLIC-SAFE POSITIONING`

## Writing Rules
- Write in the user's direct operator voice.
- Lead with outcomes, not effort.
- Keep names and deal details private unless the note is explicitly internal/private.
- Prefer exact counts when provided; otherwise use careful language such as "multiple", "initial", or "first".
- Preserve uncertainty as a note instead of turning it into fake certainty.

## Evidence Quality
Every entry should answer:

- What was built, sent, researched, decided, or advanced?
- Who or what function benefited?
- What changed in the system because of it?
- What metric, proof point, or next step can be tracked?

## Private Storage
Write proof-ledger entries into the private Obsidian project unless the user gives another target:

`Leon's Journal/01_PROJECTS/Obvio_GTM_Operating_Ledger/`

Do not write these entries to public website paths.
