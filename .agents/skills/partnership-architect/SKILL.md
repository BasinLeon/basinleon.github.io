# Partnership Architect

## Use when
Use this skill when you are talking to a potential partner (like Bounty) or a client where the deal has "Upside" components. This moves the conversation from "How much do you cost?" to "How do we share the value we create?"

## The "Upside" Modeling Protocol
When generating a partnership brief, include:
1.  **Commercial Mechanics**: Define exactly how money moves. Is it an affiliate credit? A percentage of MRR? A performance-based retainer?
2.  **The "Flywheel" Effect**: Show how the partnership makes both companies better (e.g., "Bounty provides the execution 'Muscle' while Basin::Nexus provides the 'Brain'").
3.  **The Crawl-Walk-Run Roadmap**: 
    - **Crawl**: Low-risk test (e.g., "Run 2-3 bounties").
    - **Walk**: Structured integration (e.g., "Agency-wide distribution").
    - **Run**: Automated synergy (e.g., "Nexus-to-Bounty API bridge").

## High-Status Negotiation
- **Define the Value Floor**: "Even if we only solve [X], the return is [Y]."
- **Protect the Brand**: Ensure Leon remains the "Architect" and doesn't become a "Reseller."
- **Focus on Scalability**: Pitch models that work *without* constant human intervention.

## Output Assets
- **The Partnership Brief**: A 2-page strategic document.
- **The Term Sheet Scratchpad**: A simple breakdown of the financial "Mechanics."
- **The Founder Message**: A direct, non-needy outreach to the other company's leadership.
