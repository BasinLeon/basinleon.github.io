# Human Comment Engine

## Overview

Write short comments that sound like a real operator, not an AI summary.

## When to Use

- You need a reply under a post or thread.
- The current draft feels polished, corporate, or robotic.
- You want natural phrasing, not framework-heavy writing.
- You need 1-3 fast options in different tones.

## Core Rules

- Keep comments to 1-3 lines.
- Use one clear point, not three points.
- Prefer plain language over strategy jargon.
- Avoid list format unless asked.
- No em dashes. No emoji unless user asks.
- If useful, use contrast framing: "It's not X. It's Y."

## Process

1. Identify the core point in the original post.
2. Choose one stance: agree, reframe, or gentle pushback.
3. Draft 3 variants:
- `short` (under 15 words)
- `normal` (1-2 lines)
- `spicy` (firm but respectful)
4. Remove AI markers:
- no "great point" opener
- no inflated language
- no fake certainty
5. Return the final options ready to paste.

## Output Template

- `Option 1 (short): ...`
- `Option 2 (normal): ...`
- `Option 3 (spicy): ...`

## Quality Check

A comment passes if it:

- sounds like spoken language
- can be posted without edits
- makes one memorable point
