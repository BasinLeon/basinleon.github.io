# Nexus Graphic Design Protocol

Use this skill when the user asks for "premium design," "visual architecture," or "high-status UI." This protocol ensures that every visual asset reinforces the **Revenue Architect** positioning.

## Design Principles

1. **Obsidian Foundation**: Always start with a deep dark background (`#0d0d0f` or similar). Avoid flat whites or generic grays.
2. **Glassmorphism**: Use `backdrop-filter: blur(10px)` and semi-transparent borders to create depth.
3. **Accent Glow**: Use a vibrant but focused color palette (e.g., `#3b82f6` for logic, `#a78bfa` for strategy).
4. **Structural Hierarchy**: Use high-contrast headings and JetBrains Mono for technical data points.
5. **Diagrammatic Precision**: When using Mermaid, always force a clean theme and basis curves (as seen in the Bounty-Nexus flow).

## Components

- **HUDs**: Centered layouts with status dots and infrastructure chips.
- **Node Briefs**: Clean, list-based views with diamond icons (`◈`).
- **Revenue Models**: Flowcharts that distinguish between "Commodity," "Intelligence," and "Economic" layers.

## Implementation Guide

- **Step 1**: Establish the Obsidian variable system in CSS.
- **Step 2**: Layer glass cards with `1px` translucent borders.
- **Step 3**: Inject micro-animations (hover glows and subtle shifts).
- **Step 4**: Verify typography consistency (Inter for body, Outfit for headers).
