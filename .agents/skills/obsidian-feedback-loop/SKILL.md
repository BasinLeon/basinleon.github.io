# Obsidian Feedback Loop

Use this skill whenever a task produces durable value and should be written into the Obsidian vault instead of remaining only in the chat thread.

## Default Rule
Assume meaningful work should be persisted.

That includes:

- negotiation frameworks
- strategy notes
- project operating playbooks
- new skills
- kickoff checklists
- session outcomes

## Vault Targets

### Session notes
Write session logs to:

`/Users/basin/Desktop/Basin & Associates 🌍/Leon's Journal/01_Codex Hub/01_Session Notes/`

### Reusable reference material
Write reusable frameworks and project artifacts to:

`/Users/basin/Desktop/Basin & Associates 🌍/Leon's Journal/02_Reference/`

### Business proof or case-study material
Write higher-level proof assets to:

`/Users/basin/Desktop/Basin & Associates 🌍/Leon's Journal/03_Business Cases and Accolades/`

## What To Save

### 1. The durable asset
If the session creates a reusable framework, create or update the reference note.

### 2. The session note
If the work materially changes a project or operating model, log a session note summarizing:

- context
- execution
- artifacts created
- next actions

### 3. The linking layer
Where helpful, update the relevant hub or index so future agents can find the new asset.

## Session Note Format

Use this structure:

```markdown
---
date: YYYY-MM-DD
tags:
  - ProjectName
  - Strategy
  - Skills
---

# Session - YYYY-MM-DD - Descriptive Title

## Context & Inputs

## Execution

## Artifacts Generated

## Next Actions
```

## Operator Standard

- Do not save raw chat transcripts.
- Save structured, reusable intelligence.
- Keep the writing direct and Obsidian-friendly.
- Prefer one clean note over scattered fragments.
- If a new skill is created, also log where it lives.

## Trigger Phrases

Reach for this skill when the user says things like:

- "save this to Obsidian"
- "log this"
- "make this reusable"
- "build the feedback loop"
- "capture this in the vault"

## Principle

If the insight matters enough to act on twice, it matters enough to save once.
