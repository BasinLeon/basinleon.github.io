# Artifact Generator

## Overview

Most people leave meetings with ideas. You leave with documents that move deals. This skill converts raw session notes, brainstorms, and strategy syncs into forwardable, high-status assets (Memos, Briefs, Reports).

## Core Principles

- **Forwardability**: The artifact must be clean enough for a junior (HR) to send to a senior (CEO) with zero context.
- **Outcome-Oriented**: Every artifact must have a "Next Step" or "Decision Point."
- **Asset Hierarchy**: Use markdown for speed/internal work, but prep for PDF/Web conversion for external delivery.

## Types of Artifacts

| Type | When to Use |
| :--- | :--- |
| **Diagnostic Brief** | The "Wedge" to get a call with a new lead. |
| **Strategy Memo** | Internal alignment or "Conductor" logic. |
| **Revenue Report** | Post-audit findings and architecture gaps. |
| **GTM Blueprint** | The master plan for a multi-phase engagement. |

## Process

1.  **Extract the Core Strategy**: Use `strategic-reframe` to find the headline.
2.  **Use the Template**: Select the appropriate boilerplate from the `templates/` directory.
3.  **Populate the Brief**: Fill the Context, Challenges, and Roadmap.
4.  **Confirm the Next Move**: Ensure the "Executive Close" is present.

## Integration
This is the final step in the:
`Signal → Insight → Narrative → Artifact → Deal` pipeline.
