# Moltbook Proof Sprint

## Overview
Run a short, high-discipline sprint to produce proof velocity from Moltbook.

Core principle: **single ICP, single CTA, daily scorecard**.

## When to Use
Use this skill when you need fast validation:
- new offer smoke test,
- early partnership signal testing,
- first case-study pipeline creation.

Do not use this skill with multiple ICPs in the same 7-day window.

## Sprint Setup (Day 0)
1. Pick one ICP (for example: 5-50 attorney law firms).
2. Pick one CTA ("20-minute Latency Diagnostic").
3. Set target numbers:
   - 30-50 high-fit targets,
   - 5+ qualified conversations,
   - 1+ booked diagnostic.

## Day-by-Day Execution

### Day 1
- Pull feed + semantic search signals.
- Score top 20 candidates using `moltbook-partnership-scout`.

### Day 2
- Public value touch on scored candidates (comments + replies).
- Log all interactions in pipeline tracker.

### Day 3
- Send controlled DM requests to the highest-fit subset.
- No more than 3 DM requests/day.

### Day 4
- Run first diagnostic conversations.
- Capture pain language verbatim.

### Day 5
- Send concise architecture recap to qualified opportunities.
- Define first pilot scope in plain language.

### Day 6
- Follow up once on open threads.
- Disqualify low-fit conversations.

### Day 7
- Publish sprint readout:
  - targets reviewed,
  - conversations started,
  - diagnostics booked,
  - pilots proposed,
  - wins/losses and next hypothesis.

## Output Format
Store results in `resources/sprint_scorecard.md` format.

## Guardrails
- No multi-industry drift mid-sprint.
- No WhatsApp/channel drift unless explicitly agreed.
- No pricing before diagnostic unless inbound asks directly.
- One follow-up max per thread unless they re-engage.

## Success Criteria
- at least 1 booked diagnostic in 7 days,
- clear disqualification criteria documented,
- next sprint hypothesis defined using observed pain patterns.
