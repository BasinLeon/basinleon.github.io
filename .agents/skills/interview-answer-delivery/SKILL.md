# Interview Answer Delivery

## Overview

Convert written answers into spoken answers that land in real conversations.

## When to Use

- User says: "I sound robotic" or "my answers are too long."
- User needs interview-ready delivery for hiring manager rounds.
- User has good content but weak verbal clarity.

## Answer Blueprint (Spoken)

Use this 4-part flow:

1. `Headline` (1 sentence)
2. `Proof` (2-3 sentences)
3. `Outcome` (1 sentence with metric if available)
4. `Transfer` (1 sentence: how this applies to new role)

Target length: 30-60 seconds.

## Rewrite Rules

- Replace abstract phrasing with plain verbs.
- Cut anything that sounds like a deck slide.
- Remove stacked clauses.
- Keep one metric, not five.
- End decisive, not soft.

## Delivery Markers

- Speak at 85% normal speed.
- Pause before numbers.
- Emphasize nouns and verbs, not adjectives.
- No upspeak on final sentence.

## Practice Loop

For each question:

1. Draft answer.
2. Speak it once naturally.
3. Trim 20% words.
4. Speak again with pauses.
5. Lock final version.

## Output Format

When invoked, return:

- `Question:`
- `Spoken answer (45 sec):`
- `Short answer (20 sec):`
- `Delivery note:` (pace, pause, emphasis)
