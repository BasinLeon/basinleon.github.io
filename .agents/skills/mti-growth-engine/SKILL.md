# MTI Growth Engine

Use this skill when the task involves Mock That Interview growth strategy, KPI review, onboarding analysis, outbound planning, or intern/agent delegation.

## Objective
Support MTI in moving from roughly `200-243 users` to `1,000 users in 90 days`.

Do not treat this as a pure traffic problem.

Treat it as a four-part operating problem:

1. funnel visibility
2. activation and retention
3. outbound / partnership motion
4. competitor poaching

## Before Doing Anything

Confirm or ask for the minimum access set:

- read-only product or analytics dashboard
- Google Search Console
- current website / landing pages
- outreach sheet or CRM
- any onboarding copy or product walkthrough assets

If access is missing, generate the smallest possible ask that unblocks the first diagnostic.

## Core Workflow

### 1. Establish the Baseline
Capture whatever is known now:

- total users
- weekly signup rate
- traffic sources
- first-session completion
- repeat usage or churn signal
- current outbound activity

If the data is incomplete, explicitly label the blind spots rather than inventing confidence.

### 2. Diagnose the Biggest Constraint
Pick the highest-yield bottleneck:

- poor landing-page conversion
- weak first-run experience
- low repeat usage
- unclear institutional packaging
- weak outbound targeting

Only push one or two top priorities per weekly cycle.

### 3. Convert Diagnosis into Tests
Every recommendation must become a concrete test, for example:

- rewrite intro copy on the first-run experience
- reduce friction on the interview-start flow
- create a bootcamp-specific outreach message
- build a university pilot pitch
- produce a competitor migration offer

### 4. Assign Work to the Intern Fleet
Use these roles:

#### Nexus::Funnel_Auditor
- reviews metrics
- maps drop-offs
- produces a short KPI snapshot

#### Nexus::Partnership_Prospector
- builds target lists for bootcamps, universities, and accelerators
- tags by geography and likely urgency

#### Nexus::Offer_Mapper
- turns product features into ICP-specific buying language
- drafts proof points and offer framing

#### Nexus::Retention_Scout
- inspects onboarding friction and repeat usage logic
- proposes activation experiments

### 5. Summarize in Operator Format
Output should be concise and decision-oriented:

- current bottleneck
- evidence
- proposed test
- owner
- expected signal

## Default Strategic View

### Bootcamps and career accelerators
Default near-term wedge.

Why:
- faster sales cycle
- more obvious outcome pressure
- easier to pilot

### Universities
Secondary lane.

Why:
- stronger brand upside
- slower procurement
- better for discovery and medium-term partnerships

### B2C growth
Important, but do not let random acquisition distract from activation and repeat usage.

## What Good Looks Like

- one sharp weekly KPI snapshot
- one or two focused experiments
- one prioritized target list
- one clear next ask for the founders

## What To Avoid

- fluffy "growth ideas"
- broad channel lists with no owner
- paid acquisition suggestions before activation is understood
- generic prospect lists with no fit scoring
- treating every institution as the same sale

## Reusable Phrases

- "This is not primarily a traffic problem; it is a conversion and retention architecture problem."
- "Let’s use the access handoff to build a clean baseline before we add new motion."
- "The fastest wedge is likely bootcamps and career accelerators, with universities as a slower secondary lane."
