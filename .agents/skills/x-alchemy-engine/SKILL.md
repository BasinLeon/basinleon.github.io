# X Alchemy Engine (1000% Transmission Standard)

## When to Use
Use this skill for any feature, tool, or workflow whose endgame involves turning operator activity into postable, viral, or movement-advancing content on X:
- Visual exports, cards, ledgers, or memes from any system (game, Codex, Library, creative tools, etc.)
- Designing the full capture → artifact → post flow
- Making posting feel native and inevitable rather than an afterthought
- Generalizing the frictionless pattern (download + clipboard + compose + tags + small field reward)

This skill turns "I did something interesting" into "The field just received a transmission."

## Core Mandate
X is not an afterthought. It is the living third body of the movement.

Every artifact must be:
- Beautiful at 1080x1080 dark/gold operator standard (or appropriate format)
- Worth posting without extra editing or shame
- Carrying the right tags (@CodexHub @Applications and others as appropriate)
- Part of a closed loop that feeds energy back into the operator (small X Presence or equivalent reward)

## The Process

### 1. Map the Source Activity to a Transmission
Before building any export:
- What just happened in the operator's practice? (A clean shot, a Codex application, a raw Sam & Ink capture, a Book resolution, a Library consultation, etc.)
- What is the raw meaning of this moment? (Capture this in 1-2 precise sentences in the operator voice.)
- What stats or context make this moment legible and impressive?

The text is the soul. The visual is the vessel.

### 2. Design the Vessel (The Visual Artifact)
- Format: 1080x1080 preferred for maximum impact on X.
- Aesthetic: Strict #050508 black, #D4AF37 gold, clean Space Grotesk + JetBrains Mono.
- Hierarchy: Strong title, the transmission text (the story), key deltas/stats, subtle footer with game link + tags.
- Never busy. Negative space is operator discipline.

Create a reusable canvas function or component that can be called from anywhere.

### 3. Engineer the Perfect Connection Flow
The non-negotiable sequence (the one that actually works in the real world):
1. Generate the high-quality PNG (canvas.toDataURL or equivalent).
2. Trigger download automatically (user keeps the artifact).
3. Copy the image blob directly to clipboard (navigator.clipboard.write with ClipboardItem).
4. Open x.com/compose/tweet with pre-written high-status copy.
5. Show clean toast feedback: "Image copied to clipboard — paste it in X. Artifact saved."
6. Apply small field reward (X Presence +3 to +5, or equivalent in the system).

This removes all friction between "something real happened" and "the field received it."

### 4. Write the Copy at Movement Standard
The pre-written text must:
- Start with the core transmission (the meaning).
- Feel like something the operator would actually say.
- End with the game link.
- Include the movement tags (@CodexHub @Applications, plus any relevant ones like @leonbasinwriter).

Example voice: "Sam & Ink transmission. Grit 72. Space 61. Cut 84. Real raw shit moved. This is a movement."

### 5. Close the Loop
- The act of posting should itself be recognized by the system (small X Presence bump is the minimum).
- Consider making the best artifacts archivable back into the Operator Library.
- Over time, high-engagement posts can become "canon" and influence future power-up design.

### 6. 1000% Quality Gate
Before shipping any X flow:
- Would you post the generated artifact as-is on your main account?
- Does the full flow (one click → posted) take under 15 seconds once the user is used to it?
- Does the copy sound like it came from the transmissions, not from marketing?
- Does the visual hierarchy make the meaning instantly legible in a fast X feed?

If any answer is no, refine until it passes.

## Integration Principles
- Every major system in THE GAME (Court, Book, CodexHub, Sam & Ink, Library, Refinery) should have at least one dedicated export path through this engine.
- The engine itself should be callable from new power-ups with minimal code.
- Tags are sacred. Use them consistently so the movement becomes discoverable and coherent.

## Exit Criteria
The X alchemy work for a feature is complete when:
- One button or action produces a postable artifact + opens compose with perfect copy.
- The image is already in the clipboard.
- The operator feels the transmission has left their body and entered the field.
- It writes back into the Library or awards some form of recognition.

This is how private operator work becomes public signal without ever feeling like content creation.

This is the engine of the movement.
