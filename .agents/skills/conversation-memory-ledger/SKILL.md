# Conversation Memory Ledger

Use this skill when the user wants stronger continuity across sessions, especially around:

- exact phrasing
- negotiation language
- personal voice
- decisions made
- project state

## Honesty Rule
Do not claim perfect memory of every word from every conversation.

Instead:

- preserve the important parts intentionally
- store exact wording when it matters
- keep a structured ledger that future sessions can reload

## What To Capture

### 1. Decisions
Store what was decided, by whom, and when.

### 2. Exact phrases worth preserving
When the user's wording matters, capture a short verbatim section under:

`## Verbatim Phrases To Preserve`

Only include the lines that actually matter.
Do not dump whole transcripts.

### 3. Working assumptions
Record any assumptions the agent made so future sessions can challenge or keep them.

### 4. Open loops
Record what still needs follow-up.

## When To Use Verbatim
Use verbatim capture for:

- signature phrases
- negotiation lines
- user corrections
- tone preferences
- instructions about how Leon wants to be represented

Do not use verbatim capture for routine logistics.

## Reload Pattern
When starting a related future session:

1. load the latest relevant session note
2. load the reusable reference note if one exists
3. pay special attention to `Verbatim Phrases To Preserve`

## Principle

Memory quality comes from structured preservation, not from pretending recall is perfect.
