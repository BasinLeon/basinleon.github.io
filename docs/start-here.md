# Start Here

Welcome. This site is the public thinking layer behind my portfolio and operating work.

It has two lanes:
- operator writing (GTM systems, revenue architecture, AI workflows)
- personal/literary writing (memory, reflection, and voice-led essays)

If you are new, use this path:

1. Start with the [Frameworks](/frameworks/) page.
2. Browse [Topics](/topics/) based on your role.
3. Review the [Top 20](/top-20/) priority ideas.
4. Use the [Editorial OS](/editorial-os/) to see the publish + distribution loop.
5. Explore the [Literary Lane](/literary/) for the personal side.
6. Use the [Share Kit](/share/) if you want to send this to your team.

## Who This Is For

- GTM and RevOps leaders
- Founder-led sales teams
- Operators building AI-enabled systems
- Hiring managers assessing systems execution
- Engineers and creators who want to inspect the thinking behind the work
- Readers who care about how complex work gets explained, taught, and adopted

## What You Will Get

- Practical models you can test in 7 days
- Playbooks with concrete operating steps
- Case-style writing tied to real execution decisions
- Notes that read like operator memos, class notes, and working doctrine

## Canonical Post Policy

All original full-length posts are published here first.
Cross-posts on Medium/Substack/X/LinkedIn should point back to the canonical URL on this site.
